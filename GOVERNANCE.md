# Project Governance

PhishGuard AI is an open-source security project created and led by
[Omobolaji Adeyan](https://github.com/omobolajiadeyan).

## Roles

### Lead Maintainer

The lead maintainer sets project direction, reviews security-sensitive
changes, manages releases, and makes final decisions when contributor
discussion does not reach consensus.

Current lead maintainer: **Omobolaji Adeyan**

### Contributors

Anyone may contribute through issues, discussions, pull requests,
documentation, testing, research, and review. Contributors are credited in
[AUTHORS.md](AUTHORS.md) after their first accepted pull request.

### Future Maintainers

Consistent contributors may be invited to review pull requests or maintain a
specific project area. Maintainer access is based on demonstrated judgment,
technical quality, respectful collaboration, and sustained participation.

## Decision Process

- Small fixes and documentation changes are decided through pull-request review.
- Detection-model changes require positive and negative regression evidence.
- Security-sensitive changes require approval from the lead maintainer.
- Large architectural changes should begin in GitHub Discussions or a design
  issue before implementation.

## Project Values

- Explainable security decisions
- Evidence before claims
- Reproducible testing
- Safe, synthetic test data
- Respectful and accessible collaboration
- Clear credit for contributor work
